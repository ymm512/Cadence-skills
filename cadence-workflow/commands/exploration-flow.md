# /exploration-flow - 探索流程

调用 `exploration-flow` skill 执行探索流程。
## 使用场景
- 不确定的需求（需要快速验证想法）
- 技术研究（评估技术可行性）
- 崩型开发（POC验证）
- 实验性功能（可能不会正式上线）
- 学习新技术（边做边学）
## 功能
执行探索流程， 充许迭代验证， 支持持 4 种灵活结局：
1. 转标准流程（需求明确后技术成熟）
2. 继续探索（需要更多验证）
3. 技术储备（暂时不用）
4. 记录教训（验证失败）
## 输出
- 掴程记录或- 分析报告（可能失败）
- 原型代码（探索性代码）
- 评估报告（结局建议）
## 相关命令
- `/quick-flow` - 快速开发流程（需求明确）
- `/full-flow` - 完整开发流程（复杂项目）
- `/status` - 查看进度
- `/resume` - 恢复进度
- `/checkpoint` - 创建检查点
