# /quick-flow - 快速开发流程

调用 `quick-flow` skill 执行精简的4节点快速开发流程。

## 使用场景

- 简单功能（预估 <2小时能完成）
- 需求明确（不需要 Brainstorm 探索）
- 个人项目（不需要完整文档）
- 紧急修复（需要快速上线）
- 全新模块（不涉及存量代码）

## 功能

执行精简的4节点开发流程：Requirement → Plan → Git Worktrees → Subagent Development

跳过需求探索、存量分析、设计、设计审查，适合简单功能和个人项目。

## 输出

- 需求文档（简化版）：`cadence/docs/{date}_需求文档_{功能名称}_v1.0.md`
- 实现计划（简化版）：`cadence/designs/{date}_实现计划_{功能名称}_v1.0.md`
- 代码实现 + 单元测试（覆盖率 ≥ 80%）

## 时间预估

| 复杂度 | 代码量预估 | 总时间范围 |
|--------|-----------|-----------|
| 🟢 极简单 | <200行 | 45-75分钟 |
| 🟡 简单 | 200-500行 | 75-140分钟 |

## 相关命令

- `/full-flow` - 完整开发流程（8节点，适合复杂功能）
- `/exploration-flow` - 技术探索流程（4节点，适合研究）
- `/status` - 查看进度
- `/resume` - 恢复进度
- `/checkpoint` - 创建检查点
