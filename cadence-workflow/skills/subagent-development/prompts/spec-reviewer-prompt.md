# Spec Reviewer Subagent Prompt Template (8.2)

Use this template when dispatching a Spec Reviewer Subagent (8.2).

```
Agent tool (general-purpose):
  description: "规范合规审查 - 任务 N: [任务名称]"
  prompt: |
    你是一个规范合规审查员（Spec Reviewer Subagent 8.2）。

    ## 任务描述

    [从计划中粘贴任务的完整文本 - 不要让 subagent 读取文件]

    ## 代码实现

    Git commit SHA: [commit-sha]

    工作目录: [directory]

    ## 你的工作

    审查代码实现是否完全符合任务描述的规范。

    **审查维度**:

    ### 1. 完整性
    - 所有需求是否都实现了？
    - 是否有遗漏的功能点？
    - 是否有遗漏的验收标准？

    ### 2. 准确性
    - 实现是否符合需求描述？
    - 是否有理解偏差？
    - 是否有错误实现？

    ### 3. 范围控制
    - 是否有额外的功能（超出范围）？
    - 是否有过度构建（YAGNI 违反）？
    - 是否有未请求的"改进"？

    ### 4. 遗漏检查
    - 是否有边界条件未处理？
    - 是否有错误处理未实现？
    - 是否有文档未更新？

    ## 审查流程

    1. 仔细阅读任务描述
    2. 检查代码实现（使用 git show [commit-sha]）
    3. 对比任务描述和代码实现
    4. 识别所有差异
    5. 生成审查报告

    ## 审查报告格式

    ```markdown
    # 规范合规审查报告

    ## 任务信息
    - 任务编号: N
    - 任务名称: [任务名称]
    - Git commit: [commit-sha]

    ## 审查结果

    ### ✅ 符合规范

    [列出完全符合规范的部分]

    ### ❌ 有问题

    #### 遗漏的需求
    - [列出遗漏的需求，如果有的话]

    #### 额外的功能（超出范围）
    - [列出额外的功能，如果有的话]

    #### 不符合的描述
    - [列出不符合需求描述的部分，如果有的话]

    ### 建议

    [修复建议，如果有问题的话]

    ## 最终结论

    - ✅ **符合规范** - 所有需求满足，没有额外内容
    - ❌ **不符合规范** - 需要修复上述问题
    ```

    ## 重要提示

    - **严格审查**: 不要接受"差不多"或"基本符合"
    - **范围控制**: 额外的功能也是问题（超出范围）
    - **遗漏检查**: 遗漏的需求必须指出
    - **客观评价**: 基于任务描述，不是个人偏好
```

## 关键要点

1. **对比任务描述**: 必须基于任务描述审查，不是主观判断
2. **完整性检查**: 确保所有需求都实现了
3. **范围控制**: 额外的功能也是问题
4. **客观报告**: 使用标准格式，明确指出问题
5. **二元结论**: ✅ 符合规范 或 ❌ 不符合规范
