# Phase修改遗漏文档检查模式

**创建日期**: 2026-02-28
**适用场景**: Phase修改总结验证、全局一致性检查、遗漏文档发现

---

## 核心模式

### 问题场景
Phase修改总结显示"已完成"，但实际有相关文档遗漏修改。

### 解决方案
系统性的全局验证流程，确保所有相关文档都已更新。

---

### 步骤3：全局搜索相关文档

**使用grep全局搜索**：
```bash

# 修改2：技术栈检测简化

# 修改7：Skill分类简化
grep -l "元Skill|前置Skill|支持Skill" *.md
```

---

### 步骤4：排除已知文档

**排除修改计划和总结文档**：
```bash
grep -l "关键词" *.md | grep -v "修改计划|Phase.*修改总结"
```

**为什么要排除**：
- 修改计划文档：记录了修改前的状态（包含旧术语是正常的）
- Phase总结文档：记录了修改过程（包含旧术语是正常的）
- 只关注其他文档是否还有旧术语

---

### 步骤5：逐一验证相关文档

**检查每个文档**：
1. 使用 `Read` 读取文档
2. 使用 `Grep -C 3 -n` 查看上下文
3. 判断是否需要修改

**判断标准**：
- ❌ **需要修改**：描述性内容（还在推荐旧做法）
- ✅ **无需修改**：历史性内容（说明"已废弃"、"旧版本"）
- ✅ **无需修改**：对比性内容（说明"修改前"、"修改后"）

---

### 步骤6：修改并验证

**修改遗漏文档**：
1. 使用 `Edit` 工具修改
2. 保持修改的一致性（术语、格式、结构）
3. 添加必要的说明（如"已废弃"、"参考XXX文档"）

**验证修改**：
```bash
# 再次全局搜索，确认没有遗漏
grep -l "关键词" *.md | grep -v "修改计划|Phase.*修改总结" | wc -l

# 应该返回 0
```

## 关键技巧

### 技巧1：识别遗漏的信号

**信号1**：用户发现问题
- 用户发现某个文档还有旧术语
- 说明Phase总结不够完整

**信号2**：相关文档类型
- 配置文档（如插件配置）
- 架构文档（如Skills目录结构）
- 模板文档（如Plan模板）
- 引用文档（如主文档的引用部分）

**信号3**：修改范围
- 修改了一个核心概念
- 多个文档提到这个概念
- 只修改了部分文档

---

### 技巧2：高效的全局搜索

**使用正则表达式**：
```bash
# 同时搜索多个相关术语
grep -l "术语1|术语2|术语3" *.md

# 排除已知文档
grep -l "术语" *.md | grep -v "修改计划|修改总结"

# 查看上下文
grep -C 3 -n "术语" 文件名.md
```

**搜索范围**：
- ✅ 搜索所有文档（包括未列出的）
- ✅ 排除修改计划和总结文档
- ✅ 关注描述性内容（不是历史或对比）

---

### 技巧3：保持修改一致性

**术语一致性**：
- 使用相同的替换术语
- 例如："三层检测" → "两层配置"（统一）

**格式一致性**：
- 使用相同的格式（标题、列表、表格）
- 例如：废弃说明使用相同的警告格式

**结构一致性**：
- 保持文档结构的一致性
- 例如：所有Skill分类说明使用相同的结构

## 常见陷阱

### 陷阱1：只检查列出的文件

**问题**：
Phase总结列出5个文件，只检查这5个文件

**遗漏**：
- 相关文档（如插件配置文档）
- 引用文档（如主文档的引用部分）
- 架构文档（如Skills目录结构文档）

**解决**：
全局搜索相关术语，检查所有文档

---

### 陷阱2：忘记排除修改计划文档

**问题**：
全局搜索发现修改计划文档包含旧术语

**误判**：
认为修改计划文档也需要更新

**正确理解**：
- 修改计划文档记录修改前的状态（包含旧术语是正常的）
- Phase总结文档记录修改过程（包含旧术语是正常的）
- 只关注其他文档是否还有旧术语

**解决**：
使用 `grep -v` 排除这些文档

---

### 陷阱3：修改后未再次验证

**问题**：
修改了一些文档，但没有再次全局搜索验证

**遗漏**：
- 可能还有其他文档遗漏
- 修改可能不完整

**解决**：
修改后再次全局搜索，确认返回 0

## 工具对比

| 工具 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| **Bash grep** | 快速、全局、精确 | 需要命令行 | 全局搜索、批量验证 |
| **Grep tool** | 集成在Claude Code中 | 可能返回过多结果 | 单文件或少量文件搜索 |
| **Read + 手动查找** | 直观、可控 | 慢、容易遗漏 | 小范围、精确查找 |

**推荐**：优先使用 Bash grep 进行全局验证
