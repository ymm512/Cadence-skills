# 全局验证技术模式

**创建日期**: 2026-02-27
**适用场景**: 跨多个文档的术语修改、概念替换、架构调整

---

## 核心模式

### 问题场景
当需要修改一个术语或概念时，可能在多个文档中出现，容易出现遗漏。

### 解决方案
使用系统化的全局搜索和验证流程。

---

## 标准流程

### 步骤1：发现所有相关文件
```bash
# 查找所有包含关键词的文件
grep -l "关键词1|关键词2|关键词3" *.md
```

### 步骤2：排除已知文档
```bash
# 排除修改计划、总结文档等
grep -l "关键词" *.md | grep -v "修改计划|修改总结|Phase.*修改"
```

### 步骤3：查看具体匹配内容
```bash
# 查看上下文，确认是否需要修改
grep -C 3 -n "关键词" 文件名.md
```

### 步骤4：逐一修改
- 使用 Edit 工具逐一修改
- 保持修改的一致性（术语、格式、结构）

### 步骤5：最终验证
```bash
# 确认没有遗漏（应该返回0）
grep -l "关键词" *.md | grep -v "修改计划|修改总结" | wc -l
```

## 验证清单

### 修改前

- [ ] 已排除已知文档（修改计划、总结等）
- [ ] 已使用 grep -C 3 -n 查看上下文
- [ ] 已确认哪些文件需要修改

### 修改中
- [ ] 逐一修改每个文件
- [ ] 保持术语一致性
- [ ] 保持格式一致性
- [ ] 保持结构一致性

### 修改后
- [ ] 最终验证（应该返回0）
- [ ] 确认没有遗漏
- [ ] 创建修改总结

## 关键技巧

### 技巧1：使用正则表达式
```bash
# 同时搜索多个关键词
grep -l "关键词1|关键词2|关键词3" *.md

# 排除多个模式
grep -l "关键词" *.md | grep -v "模式1|模式2|模式3"
```

### 技巧2：查看上下文
```bash
# 显示前后3行，带行号
grep -C 3 -n "关键词" 文件名.md
```

### 技巧3：统计数量
```bash
# 统计还有多少文件需要修改
grep -l "关键词" *.md | grep -v "排除" | wc -l
```

## 常见陷阱

### 陷阱1：只修改配置文件，忘记文档

**问题**: Phase 1只修改了Subagent prompt文件（8.1, 8.3），但忘记修改主文档和定义文档

**解决**: 使用全局搜索检查所有文件，不仅仅是配置文件

---

### 陷阱2：忘记版本历史

**问题**: 修改了内容，但忘记更新版本历史中的描述

**解决**: 搜索范围包括"版本历史"、"v1.0"、"v2.0"等关键词

---

### 陷阱3：忘记流程图

**问题**: 修改了文本内容，但忘记修改Mermaid流程图中的节点

**解决**: 搜索范围包括".md"文件中的流程图节点（"[ ]"格式）

## 工具对比

| 工具 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| **grep** | 快速、全局、精确 | 需要命令行 | 全局搜索、批量验证 |
| **Grep tool** | 集成在Claude Code中 | 可能返回过多结果 | 单文件或少量文件搜索 |
| **Read + 手动查找** | 直观、可控 | 慢、容易遗漏 | 小范围、精确查找 |

**推荐**: 优先使用 grep 命令进行全局验证
