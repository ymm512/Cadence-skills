- Phase 1和Phase 3修改了部分文档，但相关文档未同步更新

**原因**：
- Phase总结只列出了直接修改的文件
- 没有全局检查相关文档是否需要同步更新
- 例如：修改了Subagent调用方式，但插件配置文档还在推荐agents.json

**解决方案**：
- 使用全局搜索（grep）检查所有相关术语

- 验证时要排除修改计划和总结文档

**启示**：
- "已完成"标记需要结合全局验证
- 修改一个概念时，要检查所有提到这个概念的文档

### 发现2：相关文档的同步更新问题

**问题模式**：

```
修改点 A（在文档1）→ 相关点 B（在文档2）被遗漏
```

**案例**：

- 修改一个核心概念时，列出所有提到这个概念的文档
- 使用 `grep -l` 全局搜索
- 逐一验证每个文档是否需要更新

### 发现3：全局验证的有效模式

**标准验证流程**：
```bash
# 步骤1：发现所有相关文件
grep -l "关键词1|关键词2|关键词3" *.md

# 步骤2：排除已知文档（修改计划、总结）
grep -l "关键词" *.md | grep -v "修改计划|修改总结"

# 步骤3：逐一验证
# 使用 Read + Grep 查看具体内容

# 步骤4：修改并验证
# 修改后再次搜索，确保没有遗漏

# 步骤5：最终验证
grep -l "关键词" *.md | grep -v "修改计划|修改总结" | wc -l
# 应该返回 0
```
