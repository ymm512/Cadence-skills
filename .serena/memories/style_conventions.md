# 代码风格和约定 - Cadence-skills

## 语言规则

- **必须使用中文** - 所有与用户的交互必须使用中文
- 文档内容使用中文
- 代码中的变量名、函数名等可以使用英文

## 文档风格

### 命名规范

- 格式: `YYYY-MM-DD_文档类型_文档名称_v版本号.扩展名`
- 示例: `2025-12-03_技术方案_用户认证_v1.0.md`

### 版本号规则

- 首次创建: `v1.0`
- 小更新（错别字、格式调整）: `v1.1`、`v1.2`
- 重大更新（内容大幅修改）: `v2.0`、`v3.0`

### 文档分类

| 类型 | 路径 | 说明 |
|------|------|------|
| 需求文档 | `.claude/docs/` | PRD、产品需求、业务需求 |
| 方案设计 | `.claude/designs/` | 技术方案、架构设计、API设计 |
| README文档 | `.claude/readmes/` | 项目说明、安装指南 |
| 开发笔记 | `.claude/notes/` | 临时记录、TODO |
| 分析报告 | `.claude/analysis/` | 代码分析、调研报告 |

## 禁止行为

- 禁止在项目根目录创建文档（README.md 除外）
- 禁止在 `docs/`、`documents/`、`files/` 等目录创建文档
- 所有文档必须放在 `.claude/` 对应子目录
