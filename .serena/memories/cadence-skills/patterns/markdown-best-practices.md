# Markdown 格式最佳实践

**创建日期：** 2026-02-27  
**适用范围：** 所有 markdown 文档

## 常见问题与解决方案

### 1. Mermaid 图形格式

#### ❌ 错误：在节点中使用 emoji

```mermaid
graph LR
    A[开始] --> B[处理数据] ⭐
    B --> C[结束]
```

**问题：** emoji 会导致 Mermaid 渲染失败

#### ✅ 正确：避免在节点中使用 emoji

```mermaid
graph LR
    A[开始] --> B[处理数据]
    B --> C[结束]
```

**建议：** emoji 可以用在节点外的文本中，但不要用在节点内部

---

### 2. 代码块嵌套

#### ❌ 错误：代码块嵌套

````markdown
```markdown
# 标题

```yaml
key: value
```

```mermaid
graph LR
    A --> B
```
```
````

**问题：** markdown 不支持代码块嵌套，渲染器无法正确识别结束标记

#### ✅ 正确：提取嵌套内容为独立文档

**主文档：**
```markdown
## 输出产物

> **📋 详细模板：** [template.md](./template.md)

**文件路径：** `.claude/designs/{date}_实现计划_{功能名称}_v1.0.md`

**生成时间：** 5-40 分钟（取决于复杂度）

### 主要内容概览

1. **任务概览**：总任务数、时间估计、关键路径
2. **任务清单**：按优先级和依赖关系组织的详细任务列表
...
```

**独立模板文档（template.md）：**
```markdown
# 实现计划

## 1. 任务概览
- 总任务数：X 个
...

## 2. 任务清单

### Phase 1：基础任务（P0）
...
```

**优势：**
- 避免嵌套问题
- 主文档更简洁
- 模板可独立维护和复用

### 3. 步骤编号重复

#### ❌ 错误：步骤编号重复

```markdown
## 流程步骤

**步骤 1：** 读取技术方案

**步骤 2：** 分析功能点

**步骤 9：** 生成任务清单  <!-- 重复 -->

**步骤 9：** 验证依赖关系  <!-- 重复 -->

**步骤 10：** 输出文档
```

#### ✅ 正确：确保编号唯一且连续

```markdown
## 流程步骤

**步骤 1：** 读取技术方案

**步骤 2：** 分析功能点

...

**步骤 9：** 生成任务清单

**步骤 10：** 验证依赖关系

**步骤 11：** 输出文档
```

**建议：** 使用编辑器的自动编号功能或仔细检查

## 文档组织最佳实践

### 1. 详细模板独立维护

**原则：** 超过 100 行的模板内容应独立为单独文档

**结构：**
```
.claude/designs/
├── main-doc-v2.4.md           # 主文档（简洁）
├── subagent-definition-v1.0.md # 详细定义
└── plan-template-v1.0.md      # 纯模板
```

**引用方式：**
```markdown
> **📋 详细模板：** [template.md](./template.md)
```

### 2. 版本号规范

**独立详细文档：**
- 首次创建：v1.0
- 小更新：v1.1, v1.2
- 重大更新：v2.0

**主文档：**
- 保持版本演进：v2.3 → v2.4 → v2.5

### 3. 文档命名规范

**格式：** `YYYY-MM-DD_文档类型_文档名称_v版本号.扩展名`

**示例：**
```
2026-02-26_技术方案_Subagent定义_v1.0.md
2026-02-26_输出产物_Plan实现计划模板_v1.0.md
2026-02-25_技术方案_使用Claude_Code_Skills的AI自动化开发方案_v2.4.md
```

## 代码块类型选择

### 常用代码块类型

| 类型 | 用途 | 示例 |
|------|------|------|
| `markdown` | Markdown 示例 | 文档结构示例 |
| `yaml` | YAML 配置 | tech_stack 配置 |
| `mermaid` | Mermaid 图表 | 流程图、依赖图 |
| `bash` | Shell 命令 | git 命令、构建命令 |
| `python` | Python 代码 | 代码示例 |
| `javascript` | JavaScript 代码 | 代码示例 |
| `json` | JSON 数据 | 配置文件示例 |

### 选择原则

- ✅ 使用准确的代码块类型（语法高亮）
- ✅ 如果没有特定类型，使用 `text` 或 `plaintext`
- ❌ 不要为了嵌套而使用 `markdown` 代码块

- [ ] 文档命名符合规范
