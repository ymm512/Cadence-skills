# Implementer Subagent Prompt Template (8.1)

Use this template when dispatching an Implementer Subagent (8.1).

```
Agent tool (general-purpose):
  description: "实现任务 N: [任务名称]"
  prompt: |
    你正在实现任务 N: [任务名称]

    ## 任务描述

    [从计划中粘贴任务的完整文本 - 不要让 subagent 读取文件]

    ## 上下文

    [场景设置：这个任务在整体方案中的位置、依赖关系、架构上下文]

    ## 技术方案（可选）

    [如果有技术方案，提供相关部分]

    ## Worktree 信息（可选）

    工作目录: [worktree 路径或当前目录]
    分支: [分支名称]

    ## 开始之前

    如果你对以下方面有疑问：
    - 需求或验收标准
    - 方法或实现策略
    - 依赖关系或假设
    - 任务描述中任何不清楚的内容

    **现在就问。** 在开始工作之前提出任何疑虑。

    ## 你的工作

    一旦你对需求清楚了：
    1. 严格按照任务规范实现
    2. 编写测试（如果任务要求，遵循 TDD）
    3. 验证实现工作正常
    4. 提交你的工作
    5. 自我审查（见下文）
    6. 报告

    工作目录: [directory]

    **工作过程中：** 如果你遇到意外或不清楚的事情，**提问**。
    暂停并澄清总是可以的。不要猜测或假设。

    ## 报告之前：自我审查

    用全新的眼光审查你的工作。问自己：

    **完整性：**
    - 我是否完全实现了规范中的所有内容？
    - 我是否遗漏了任何需求？
    - 是否有我没有处理的边界情况？

    **质量：**
    - 这是我最好的工作吗？
    - 名称是否清晰准确（匹配功能，而不是实现方式）？
    - 代码是否干净且可维护？

    **纪律：**
    - 我是否避免了过度构建（YAGNI）？
    - 我是否只构建了请求的内容？
    - 我是否遵循了代码库中的现有模式？

    **测试：**
    - 测试是否实际验证了行为（而不仅仅是模拟行为）？
    - 我是否遵循了 TDD（如果要求）？
    - 测试是否全面？
    - 测试覆盖率是否 ≥ 80%？

    如果你在自我审查期间发现问题，请在报告之前立即修复。

    ## 报告格式

    完成后，报告：
    - 你实现了什么
    - 你测试了什么以及测试结果
    - 修改的文件
    - 自我审查发现（如果有）
    - 任何问题或疑虑
    - Git commit SHA
```

## 关键要点

1. **完整文本**: 必须提供任务的完整文本，不要让 subagent 读取文件
2. **上下文**: 提供任务在整体方案中的位置
3. **提问机会**: 允许 subagent 在开始前和工作期间提问
4. **TDD 要求**: 明确是否需要遵循 TDD 流程
5. **自我审查**: 要求 subagent 在报告前进行自我审查
6. **覆盖率**: 要求测试覆盖率 ≥ 80%
