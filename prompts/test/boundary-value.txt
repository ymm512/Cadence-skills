## 边界值测试定义

边界值测试 (Boundary Value Testing) 是一种测试技术,专门测试:
- 输入值的边界条件
- 等价类的边界点
- 临界值附近的系统行为

**原理**: 大多数错误发生在边界附近,而非区间中间。

---

## 边界值分析方法

### 1. 数值范围边界
```yaml
for_range: "[min, max]"
test_points:
  - min - 1:     "最小值减 1 (无效)"
  - min:         "最小值 (有效)"
  - min + 1:     "最小值加 1 (有效)"
  - typical:     "典型值 (有效)"
  - max - 1:     "最大值减 1 (有效)"
  - max:         "最大值 (有效)"
  - max + 1:     "最大值加 1 (无效)"
```

### 2. 长度边界
```yaml
for_length: "[minLen, maxLen]"
test_points:
  - length_0:        "空字符串"
  - length_minLen-1: "最小长度减 1"
  - length_minLen:   "最小长度"
  - length_typical:  "典型长度"
  - length_maxLen:   "最大长度"
  - length_maxLen+1: "最大长度加 1"
```

### 3. 特殊值
```yaml
special_values:
  - null:          "空值"
  - undefined:     "未定义"
  - empty_string:  "空字符串"
  - whitespace:    "只有空格"
  - zero:          "零值"
  - negative:      "负值"
  - max_int:       "最大整数"
  - min_int:       "最小整数"
  - infinity:      "无穷大"
  - NaN:           "非数字"
```

## 常见字段类型边界

### 字符串字段
```yaml
username:
  constraints:
    min_length: 3
    max_length: 20
    pattern: "^[a-zA-Z0-9_]+$"
  boundary_tests:
    - {length: 0, value: "", expected: "invalid", error: "不能为空"}
    - {length: 2, value: "ab", expected: "invalid", error: "至少 3 个字符"}
    - {length: 3, value: "abc", expected: "valid"}
    - {length: 20, value: "a" * 20, expected: "valid"}
    - {length: 21, value: "a" * 21, expected: "invalid", error: "最多 20 个字符"}
    - {value: "user@name", expected: "invalid", error: "只能包含字母数字下划线"}

email:
  constraints:
    max_length: 255
    format: "email"
  boundary_tests:
    - {value: "", expected: "invalid", error: "不能为空"}
    - {value: "a@b.c", expected: "valid", note: "最短有效邮箱"}
    - {value: "a" * 243 + "@example.com", expected: "valid", note: "255 字符"}
    - {value: "a" * 244 + "@example.com", expected: "invalid", error: "超过最大长度"}
    - {value: "not-email", expected: "invalid", error: "邮箱格式无效"}

password:
  constraints:
    min_length: 8
    max_length: 32
    pattern: "至少包含大小写字母、数字、特殊字符"
  boundary_tests:
    - {length: 7, value: "Abc123!", expected: "invalid", error: "至少 8 个字符"}
    - {length: 8, value: "Abc1234!", expected: "valid"}
    - {length: 32, value: "Abc12345!" + "x" * 23, expected: "valid"}
    - {length: 33, value: "Abc12345!" + "x" * 24, expected: "invalid", error: "最多 32 个字符"}
    - {value: "abcdefgh", expected: "invalid", error: "需要包含大写字母"}
    - {value: "ABCDEFGH", expected: "invalid", error: "需要包含小写字母"}
    - {value: "Abcdefgh", expected: "invalid", error: "需要包含数字"}
    - {value: "Abcd1234", expected: "invalid", error: "需要包含特殊字符"}
```

### 数值字段
```yaml
age:
  constraints:
    min: 0
    max: 150
    type: integer
  boundary_tests:
    - {value: -1, expected: "invalid", error: "不能为负数"}
    - {value: 0, expected: "valid"}
    - {value: 1, expected: "valid"}
    - {value: 149, expected: "valid"}
    - {value: 150, expected: "valid"}
    - {value: 151, expected: "invalid", error: "不能超过 150"}
    - {value: 18.5, expected: "invalid", error: "必须是整数"}

price:
  constraints:
    min: 0.01
    max: 999999.99
    precision: 2
  boundary_tests:
    - {value: 0, expected: "invalid", error: "必须大于 0"}
    - {value: 0.01, expected: "valid"}
    - {value: 0.001, expected: "invalid", error: "最多两位小数"}
    - {value: 999999.99, expected: "valid"}
    - {value: 1000000, expected: "invalid", error: "超过最大金额"}
    - {value: -0.01, expected: "invalid", error: "不能为负数"}

quantity:
  constraints:
    min: 1
    max: 9999
    type: integer
  boundary_tests:
    - {value: 0, expected: "invalid", error: "至少为 1"}
    - {value: 1, expected: "valid"}
    - {value: 9999, expected: "valid"}
    - {value: 10000, expected: "invalid", error: "超过最大数量"}
```

### 日期字段
```yaml
birth_date:
  constraints:
    min: "1900-01-01"
    max: "today"
  boundary_tests:
    - {value: "1899-12-31", expected: "invalid", error: "日期太早"}
    - {value: "1900-01-01", expected: "valid"}
    - {value: "today", expected: "valid"}
    - {value: "tomorrow", expected: "invalid", error: "不能是未来日期"}
    - {value: "invalid-date", expected: "invalid", error: "日期格式无效"}

appointment_date:
  constraints:
    min: "today"
    max: "today + 90 days"
  boundary_tests:
    - {value: "yesterday", expected: "invalid", error: "不能是过去日期"}
    - {value: "today", expected: "valid"}
    - {value: "today + 90 days", expected: "valid"}
    - {value: "today + 91 days", expected: "invalid", error: "不能超过 90 天"}
```

### 枚举字段
```yaml
status:
  constraints:
    allowed_values: ["pending", "active", "inactive", "deleted"]
  boundary_tests:
    - {value: "pending", expected: "valid"}
    - {value: "active", expected: "valid"}
    - {value: "unknown", expected: "invalid", error: "无效的状态值"}
    - {value: "", expected: "invalid", error: "状态不能为空"}
    - {value: "PENDING", expected: "invalid", error: "状态值区分大小写"}
```

### 数组字段
```yaml
tags:
  constraints:
    min_items: 0
    max_items: 10
    item_max_length: 20
  boundary_tests:
    - {value: [], expected: "valid"}
    - {value: ["tag1"], expected: "valid"}
    - {value: Array(10).fill("tag"), expected: "valid"}
    - {value: Array(11).fill("tag"), expected: "invalid", error: "最多 10 个标签"}
    - {value: ["a" * 21], expected: "invalid", error: "标签最多 20 个字符"}
```

## 组合边界测试

### 两个字段组合
```yaml
password_confirmation:
  fields: ["password", "confirm_password"]
  boundary_tests:
    - {password: "Abc12345!", confirm: "Abc12345!", expected: "valid"}
    - {password: "Abc12345!", confirm: "Abc12345?", expected: "invalid", error: "密码不匹配"}
    - {password: "", confirm: "", expected: "invalid", error: "密码不能为空"}

date_range:
  fields: ["start_date", "end_date"]
  boundary_tests:
    - {start: "2026-01-01", end: "2026-01-02", expected: "valid"}
    - {start: "2026-01-01", end: "2026-01-01", expected: "valid", note: "同一天"}
    - {start: "2026-01-02", end: "2026-01-01", expected: "invalid", error: "结束日期不能早于开始日期"}
```

## 输出格式

返回 JSON 格式的边界值测试用例:

```json
[
  {
    "test_case_id": "BV-001",
    "name": "字段名 - 边界描述",
    "priority": "P1",
    "type": "boundary_value",
    "field": "字段名",
    "constraint": "约束描述",
    "description": "测试目的描述",
    "test_data_set": [
      {
        "value": "测试值",
        "boundary": "min|min-1|min+1|max|max-1|max+1|special",
        "expected_valid": true|false,
        "expected_error": "错误消息|null"
      }
    ],
    "automation_hint": "自动化提示"
  }
]
```

每个字段生成:
- 至少 6 个边界点 (min-1, min, min+1, max-1, max, max+1)
- 适用的特殊值测试 (null, empty, etc.)
- 格式验证测试 (如果有 pattern 约束)
