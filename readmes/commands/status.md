# /status Command

## 概述

`/status` 命令用于查看当前开发进度，显示当前流程、节点、已完成任务和下一步操作。

## 如何使用

```bash
/status
```

## 显示内容

### 基本信息

- **流程类型**：Full Flow / Quick Flow / Exploration Flow
- **当前节点**：节点编号和名称
- **预估剩余时间**：基于历史数据估算

### 已完成节点

- 节点名称和描述
- 生成的文档路径
- 完成时间

### 当前节点

- 节点状态（进行中/暂停）
- 预估时间
- 已用时间
- 当前任务（如果有）

### 待完成节点

- 节点列表
- 预估时间

### 进度统计

- 总节点数
- 已完成百分比
- 进行中百分比
- 待完成百分比

## 相关命令

- `/resume` - 恢复进度
- `/checkpoint` - 创建检查点
- `/report` - 生成详细报告
- `/monitor` - 实时监控

## 最佳实践

### 1. 定期检查进度

每个节点完成后使用 `/status` 查看进度。

### 2. 结合检查点使用

```bash
/status           # 查看进度
/checkpoint       # 创建检查点
```

### 3. 恢复前先查看

```bash
/status           # 查看当前进度
/resume           # 恢复进度
```
