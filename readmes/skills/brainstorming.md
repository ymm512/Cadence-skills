# brainstorming Skill

## 概述

`brainstorming` 是需求探索 Skill，通过苏格拉底式对话明确需求，探索替代方案，分段展示设计供验证。

## 如何单独使用

### 命令调用

```bash
/brainstorm
```

### 自动触发

当用户说以下内容时自动触发：
- "帮我实现..."
- "我想构建..."
- "让我们开发..."
- "需要添加..."

## 具体使用案例

### 案例 1：新功能开发

**用户输入**：
```
帮我实现用户登录功能
```

**执行流程**：
1. 🎯 **理解目标** - 通过提问明确真正想要什么
   ```
   我理解您想实现用户登录功能。让我问几个问题来更好地理解需求：

   1. 这是什么类型的应用？（Web/移动端/API）
   2. 需要支持哪些登录方式？（用户名密码/社交登录/手机号）
   3. 有什么特殊的认证需求吗？（2FA/SSO/JWT）
   ```

2. 🔄 **探索替代方案** - 提供不同的实现选项
   ```
   基于您的需求，我看到几种实现方案：

   方案 A：传统的用户名密码登录
   - 优点：简单、用户熟悉
   - 缺点：安全性较低、需要密码管理

   方案 B：OAuth 2.0 社交登录
   - 优点：用户体验好、安全性高
   - 缺点：依赖第三方服务

   方案 C：手机号 + 验证码
   - 优点：便捷、安全
   - 缺点：需要短信服务、成本较高

   您倾向于哪种方案？
   ```

3. 📋 **分段展示设计** - 分块展示设计，每块都可验证
   ```
   好的，基于您选择的 OAuth 2.0 方案，让我分段展示设计：

   ## 第一部分：技术选型

   - 认证框架：Passport.js（Node.js）
   - OAuth Provider：Google, GitHub
   - Session 管理：JWT Token

   这部分看起来合理吗？还是需要调整？
   ```

4. 💾 **保存设计文档** - 输出到 `.claude/designs/`
   ```
   设计已确认，正在保存设计文档到：
   .claude/designs/2026-03-02_技术方案_用户登录功能_v1.0.md
   ```

### 案例 2：技术选型

**用户输入**：
```
我应该使用 GraphQL 还是 REST？
```

**执行流程**：
1. 探索项目需求
2. 对比两种方案
3. 给出建议和理由
4. 保存决策文档

## 输出格式

### 需求文档

```markdown
# {功能名称} 需求文档

## 背景
{项目背景}

## 目标
{明确的目标}

## 用户故事
- 作为 {角色}，我想要 {功能}，以便 {价值}

## 技术方案
{选定的技术方案}

## 替代方案
{考虑过的其他方案}

## 风险和限制
{已知的风险和限制}

## 下一步
- [ ] 需求分析
- [ ] 技术设计
- [ ] 实现计划
```

## 最佳实践

### 1. 不要假设需求

通过提问明确需求，而不是假设你知道用户想要什么。

### 2. 探索多种方案

总是提供至少 2-3 种替代方案，并说明优缺点。

### 3. 分段验证

不要一次性展示整个设计，而是分段展示，每段都确认。

### 4. 保存文档

将最终设计保存到 `.claude/designs/` 目录，便于后续参考。

### 5. 引导到下一步

完成 Brainstorm 后，引导用户进入下一个节点（Analyze 或 Requirement）。

## 常见问题

### Q: Brainstorm 和 Requirement 有什么区别？

**Brainstorm**：
- 目标：探索需求、明确目标
- 输出：需求文档（轻量级）
- 重点：探索替代方案

**Requirement**：
- 目标：详细分析需求
- 输出：结构化需求文档（详细）
- 重点：功能点、边界条件、测试用例

### Q: 什么时候使用 Brainstorm？

- ✅ 新功能开发
- ✅ 技术选型
- ✅ 探索性任务
- ✅ 不确定的需求

### Q: 什么时候跳过 Brainstorm？

- ❌ 简单的 Bug 修复 → 使用 Quick Flow
- ❌ 明确的小改动 → 直接进入 Plan
- ❌ 紧急修复 → 使用 Quick Flow

## 相关 Skills

- **analyze** - 存量分析（通常在 Brainstorm 之后）
- **requirement** - 需求分析（Brainstorm 的详细版本）
- **design** - 技术设计
- **exploration-flow** - 探索流程（包含 Brainstorm）

## 流程集成

### 完整流程（Full Flow）

```
Brainstorm → Analyze → Requirement → Design → Design Review → Plan → Worktree → Develop
```

### 探索流程（Exploration Flow）

```
Brainstorm → Analyze → Design → Plan → (4 种结束方式)
```

### 单独使用

```
Brainstorm → (保存文档) → (用户决定下一步)
```
