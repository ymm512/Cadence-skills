## 📋 使用指南

### 标准流程（获取锁 → 执行操作 → 释放锁）

```markdown
# Step 1: 准备参数
resource_name = "progress-{project_id}"
task_id = "checkpoint-{phase}"
ttl = 300  # 5分钟

# Step 2: 获取锁
调用 lock-utils skill (Acquire Lock)

IF 获取锁失败:
  → 记录错误日志
  → 返回失败

# Step 3: 执行关键操作
try:
  3.1 保存 Checkpoint
  3.2 更新 Progress
  3.3 更新索引
  → 所有操作成功

except 操作失败:
  → 记录错误日志
  → 返回失败

# Step 4: 释放锁 (finally)
finally:
  调用 lock-utils skill (Release Lock)
```

## ⚠️ 注意事项

### 关键警告

🔴 **必须验证锁**: 获取锁后必须验证，防止并发竞争

🔴 **必须释放锁**: 无论操作成功或失败，都必须释放锁

🔴 **只有持有者可释放**: 不是锁的持有者不能释放锁

### 使用建议

✅ **安全场景**: 单用户、单会话、顺序操作（完全安全）

✅ **谨慎使用**: 多会话、并发操作（**已支持**，但需遵循锁使用规范）

⚠️ **需要测试**: 高频并发写入场景（需要实际测试验证锁机制）
