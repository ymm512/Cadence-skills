## 🖥️ `/monitor` 功能重新定位

### 问题背景

**原始设计**:

```yaml
原始设计:
  功能: 实时监控项目状态
  特性:
    - 每5秒自动刷新
    - 实时监听任务状态变更
    - 实时计算性能指标
    - 实时显示进度条

  问题:
    ❌ Claude Code 不支持后台任务
    ❌ Claude Code 不支持定时器
    ❌ Claude Code 不支持事件监听
    ❌ 无法实现"实时"功能
```

---

### 新设计

#### 从"实时监控"改为"状态快照"

```yaml
新设计:
  功能: 查看当前状态快照
  特性:
    - 一次性快照,不自动刷新
    - 查询时读取当前状态
    - 查询时计算指标
    - 手动刷新(再次执行命令)

  优势:
    ✅ 符合 Claude Code 技术限制
    ✅ 功能完全可实现
    ✅ 用户期望清晰
```

# /monitor - 查看当前状态快照

## 使用场景

- 查看项目当前状态
- 了解任务执行进度
- 检查性能指标

## 调用方式

## Overview

**Core Principle**: 查看项目的当前状态快照,一次性显示当前进度。

## When to Use

**Use this skill when:**
- 需要查看项目当前状态
- 需要了解任务执行进度

## The Process

### Step 1: 获取当前项目ID

读取当前项目ID:

```yaml
project_id = get_project_id()
```

### Step 2: 读取进度数据

读取项目的 Progress 数据:

```yaml
progress_data = read_memory(f"progress-{project_id}")
```

### Step 3: 读取当前任务

读取 TodoWrite 的当前任务:

```yaml
current_tasks = get_todowrite_tasks()
```

### Step 4: 计算性能指标

计算时间统计:

```yaml
time_stats = calculate_time_stats(progress_data)
```

### Step 5: 格式化输出

格式化状态快照显示:

```yaml
display_status_snapshot(progress_data, current_tasks, time_stats)
```

## Quick Reference

| Step | Action | Tool |
|------|--------|------|
| 1 | 获取项目ID | `get_project_id()` |
| 2 | 读取进度数据 | `read_memory()` |
| 3 | 读取当前任务 | `get_todowrite_tasks()` |
| 4 | 计算性能指标 | `calculate_time_stats()` |
| 5 | 格式化输出 | `display_status_snapshot()` |

## Code Example

```python
def monitor_skill():
    """
    查看项目当前状态快照
    """
    # Step 1: 获取当前项目ID
    project_id = get_project_id()

    # Step 2: 读取进度数据
    progress_data = read_memory(f"progress-{project_id}")

    if not progress_data:
        print("❌ 未找到项目进度数据")
        return

    # Step 3: 读取当前任务
    current_tasks = get_todowrite_tasks()

    # Step 4: 计算性能指标
    time_stats = calculate_time_stats(progress_data)

    # Step 5: 格式化输出
    print("\n📊 项目状态快照")
    print(f"⏰ 快照时间: {get_current_timestamp()}\n")

    print("## 当前任务")
    for task in current_tasks:
        if task.status == "in_progress":
            print(f"- **{task.subject}**")
            print(f"  - 状态: 进行中")
            print(f"  - 开始时间: {task.start_time}")
            print(f"  - 已耗时: {calculate_elapsed_time(task.start_time)}")
            print(f"  - 重试次数: {task.retry_count}")

    print("\n## 整体进度")
    percentage = progress_data.overall_progress.percentage
    completed = progress_data.overall_progress.completed_phases
    total = progress_data.overall_progress.total_phases

    progress_bar = generate_progress_bar(percentage)
    print(f"{progress_bar} {percentage:.1f}% ({completed}/{total} 节点)")

    print("\n## 时间统计")
    print(f"- 已用时: {format_duration(time_stats['total_time'])}")
    print(f"- 预估剩余: {format_duration(time_stats['estimated_remaining'])}")

    print("\n---")
    print("💡 提示: 这是一个快照,不会自动刷新。如需更新,请再次执行 /monitor\n")
```

## Common Mistakes

### ❌ 期望自动刷新

```yaml
# 错误期望
用户期望: /monitor 会每5秒自动刷新

实际情况: /monitor 只显示一次性快照,不会自动刷新

解决方案: 在文档中明确说明,并在输出中提示用户
```

### ❌ 期望实时监听

```yaml
# 错误期望
用户期望: 任务状态变更时自动更新

实际情况: 需要手动再次执行 /monitor 才能看到更新

解决方案: 在文档中明确说明"一次性快照"的特性
```

## Red Flags

**STOP if:**
- 尝试实现自动刷新功能
- 尝试实现事件监听机制
- 尝试实现定时器功能

以上功能在 Claude Code 中**不可行**。

## 🚫 移除不切实际的设计假设

### 设计假设清单

#### 1. 自动刷新机制

```yaml
原设计假设:
  - 每5秒自动刷新
  - 后台定时任务
  - 无需用户干预

实际情况:
  - Claude Code 不支持后台任务
  - Claude Code 不支持定时器
  - 需要手动刷新

调整方案:
  - 改为一次性快照
  - 手动刷新(再次执行命令)
  - 在输出中明确提示
```

---

#### 2. 实时事件监听

```yaml
原设计假设:
  - 实时监听任务状态变更
  - 事件驱动更新
  - 自动响应变化

实际情况:
  - Claude Code 不支持事件监听
  - Claude Code 不支持回调机制
  - 需要主动查询

调整方案:
  - 改为查询时读取状态
  - 查询时计算指标
  - 手动触发更新
```

---

#### 3. 性能指标实时更新

```yaml
原设计假设:
  - 实时计算性能指标
  - 自动更新进度条
  - 动态显示统计

实际情况:
  - 无法实现实时更新
  - 只能在查询时计算
  - 需要手动刷新

调整方案:
  - 查询时计算指标
  - 一次性显示统计
  - 手动刷新获取最新数据
```

---

#### 4. 会话中断前自动保存

```yaml
原设计假设:
  - 会话中断前自动保存
  - 检测会话结束事件
  - 自动创建 Checkpoint

实际情况:
  - 无法检测会话中断
  - Claude Code 不支持会话结束事件
  - 无法执行自动保存

调整方案:
  - 节点完成后立即保存
  - 每个节点完成时创建 Checkpoint
  - 不依赖会话中断检测
```
