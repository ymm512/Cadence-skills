## 📋 使用指南

### 标准流程（开始事务 → 执行操作 → 提交/回滚）

```markdown
# Step 1: 识别关键资源
critical_resources = [
  "progress-{project_id}",
  "index-{project_id}-checkpoints-by-time",
  "index-{project_id}-checkpoints-by-phase"
]

# Step 2: 开始事务（创建备份）
调用 transaction-utils skill (Begin Transaction)

IF 备份创建失败:
  → 记录错误日志
  → 返回失败

# Step 3: 执行关键操作
try:
  3.1 保存 Checkpoint
  3.2 更新 Progress
  3.3 更新索引
  → 所有操作成功

except 操作失败:
  → 记录错误日志
  → 触发回滚

# Step 4: 提交或回滚
IF 所有操作成功:
  → 提交事务（清理备份）
IF 任何操作失败:
  → 回滚事务（恢复备份）
```

## ⚠️ 注意事项

### 关键警告

🔴 **必须备份所有关键资源**: 只备份部分会导致回滚不完整

🔴 **必须验证备份**: 不验证可能导致回滚时发现备份不存在

🔴 **回滚顺序很重要**: 反向回滚，先恢复被依赖的资源

🔴 **区分新创建和已存在**: 新创建资源回滚时删除，已存在资源回滚时恢复

### 使用建议

✅ **完全支持**: 单用户、单会话、顺序操作（完全安全）

✅ **完全支持**: 多会话、并发操作（**已支持**，锁+事务机制已就位）
