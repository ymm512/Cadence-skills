## 代码要求

### 1. 代码规范
- 遵循项目已有的代码风格
- 使用 TypeScript 进行类型注解 (如果适用)
- 组件命名使用 PascalCase
- 变量和函数命名使用 camelCase
- 常量命名使用 UPPER_SNAKE_CASE

### 2. 组件设计
- **单一职责**: 组件只做一件事
- **可复用性**: 组件可在不同场景使用
- **Props 定义**: 明确所有 Props 的类型和用途
- **默认值**: 为 Props 提供合理的默认值
- **文档注释**: 为组件和复杂逻辑添加注释

### 3. 状态管理
- 优先使用局部状态 (useState)
- 复杂状态使用 useReducer
- 全局状态使用 Redux/Zustand (根据项目配置)
- 避免不必要的状态提升

### 4. 性能优化
- 使用 React.memo 包裹纯组件
- 使用 useMemo 缓存计算��果
- 使用 useCallback 缓存回调函数
- 避免在 render 中创建新对象/数组
- 合理使用 key 属性

### 5. 错误处理
- 使用 Error Boundary 捕获组件错误
- 对异步操作进行错误处理
- 提供友好的错误提示
- 避免应用崩溃

### 6. 可访问性 (A11y)
- 使用语义化 HTML 标签
- 添加 aria-label 和 aria-describedby
- 支持键盘导航
- 提供合适的 focus 样式

```typescript

  const [loading, setLoading] = useState(true);
  const [data, setData] = useState(null);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetchData()
      .then(setData)
      .catch(setError)
      .finally(() => setLoading(false));
  }, []);

  if (loading) return <LoadingSpinner />;
  if (error) return <ErrorMessage error={error} />;
  return <DataDisplay data={data} />;
```
