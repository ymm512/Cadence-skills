## Git 分支策略

### 分支命名规范
```yaml
feature:
  pattern: "feature/{ticket}-{description}"
  example: "feature/WF-20260209-user-auth"

bugfix:
  pattern: "bugfix/{ticket}-{description}"
  example: "bugfix/BUG-123-login-fix"

hotfix:
  pattern: "hotfix/{version}-{description}"
  example: "hotfix/v1.2.1-security-patch"

release:
  pattern: "release/{version}"
  example: "release/v1.3.0"
```

### 分支生命周期
```
main (稳定版本)
  └── develop (开发主线)
        ├── feature/xxx (功能开发)
        ├── bugfix/xxx (Bug 修复)
        └── release/xxx (发布准备)
              └── hotfix/xxx (紧急修复)
```

## Commit Message 规范

### 格式
```
<type>(<scope>): <subject>

<body>

<footer>
```

### Type 类型
```yaml
feat:
  description: "新功能"
  example: "feat(auth): 添加用户登录功能"

fix:
  description: "Bug 修复"
  example: "fix(login): 修复密码验证错误"

docs:
  description: "文档更新"
  example: "docs(readme): 更新安装说明"

style:
  description: "代码格式 (不影响功能)"
  example: "style(components): 格式化代码"

refactor:
  description: "重构 (非新功能非修复)"
  example: "refactor(user): 重构用户服务"

perf:
  description: "性能优化"
  example: "perf(query): 优化数据库查询"

test:
  description: "测试相关"
  example: "test(auth): 添加登录测试用例"

chore:
  description: "构建/工具相关"
  example: "chore(deps): 升级依赖版本"
```

### Scope 范围 (可选)
```yaml
common_scopes:
  - auth: "认证模块"
  - user: "用户模块"
  - order: "订单模块"
  - api: "API 接口"
  - ui: "界面组件"
  - db: "数据库"
  - config: "配置"
  - deps: "依赖"
```

### Subject 规范
- 使用祈使句 (动词原形开头)
- 不超过 50 个字符
- 首字母小写 (中文除外)
- 不使用句号结尾

### Body 规范
- 说明 **为什么** 做这个修改
- 说明 **如何** 解决问题
- 每行不超过 72 个字符

### Footer 规范
```yaml
breaking_change:
  format: "BREAKING CHANGE: <description>"
  example: "BREAKING CHANGE: API 响应格式变更"

issue_reference:
  format: "Closes #<issue_number>"
  example: "Closes #123"

co_author:
  format: "Co-Authored-By: <name> <email>"
  example: "Co-Authored-By: Claude <noreply@anthropic.com>"
```

## Commit Message 示例

### 功能开发
```
feat(auth): 实现用户登录功能

- 添加登录 API 端点 POST /api/auth/login
- 实现 JWT token 生成和验证
- 添加密码加密和验证逻辑
- 集成 Redis 存储 refresh token

业务规则:
- 密码必须包含大小写字母、数字和特殊字符
- 登录失败 5 次锁定账户 30 分钟
- Token 有效期 15 分钟

Closes #WF-20260209-abc123

Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>
```

### Bug 修复
```
fix(login): 修复登录验证逻辑错误

问题:
- 用户输入正确密码但提示"密码错误"

原因:
- bcrypt.compare 参数顺序错误
- 应该是 compare(plainPassword, hashedPassword)
- 错误写成 compare(hashedPassword, plainPassword)

修复:
- 修正 bcrypt.compare 参数顺序
- 添加单元测试防止回归

Closes #BUG-456

Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>
```

### 重构
```
refactor(user): 重构用户服务为分层架构

将 UserService 重构为三层架构:
- Controller: HTTP 请求处理
- Service: 业务逻辑
- Repository: 数据访问

改动:
- 新增 UserController.ts
- 新增 UserRepository.ts
- 修改 UserService.ts (移除数据访问逻辑)
- 更新相关测试

无功能变更,仅结构调整。

Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>
```

## Git 操作流程

### 1. 创建分支
```bash
# 从 main/develop 创建 feature 分支
git checkout main
git pull origin main
git checkout -b feature/WF-{workflow_id}-{description}
```

### 2. 提交变更
```bash
# 查看变更
git status
git diff

# 暂存变更 (推荐分文件添加)
git add src/services/UserService.ts
git add src/controllers/UserController.ts

# 提交 (使用 HEREDOC 格式化多行消息)
git commit -m "$(cat <<'EOF'
feat(auth): 实现用户登录功能

详细描述...

Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>
EOF
)"
```

### 3. 推送到远程
```bash
# 首次推送 (设置上游)
git push -u origin feature/WF-{workflow_id}

# 后续推送
git push
```

### 4. 创建 Pull Request
```bash
# 使用 gh CLI 创建 PR
gh pr create \
  --title "feat(auth): 实现用户登录功能" \
  --body "$(cat <<'EOF'
## Summary
- 实现用户登录 API
- 添加 JWT 认证
- 完成单元测试

## Test Plan
- [ ] 运行单元测试
- [ ] 手动测试登录流程
- [ ] 代码审查

## Related Issues
Closes #WF-20260209-abc123

🤖 Generated with Claude Code
EOF
)" \
  --base main
```

## 安全注意事项

### 禁止操作
```yaml
never_do:
  - "git push --force (除非明确需要)"
  - "git reset --hard origin/main"
  - "修改已 push 的 commit"
  - "提交敏感信息 (.env, credentials)"
  - "跳过 hooks (--no-verify)"
```

### 检查清单
```yaml
before_commit:
  - "git diff 审查变更"
  - "确保没有敏感信息"
  - "确保测试通过"
  - "确保代码格式正确"

before_push:
  - "git log 检查 commit 历史"
  - "确认目标分支正确"
  - "确认无未提交变更"
```
