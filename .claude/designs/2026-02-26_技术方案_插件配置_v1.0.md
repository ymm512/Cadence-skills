# 插件配置设计文档

> **文档版本**: v1.0
> **创建日期**: 2026-02-26
> **所属方案**: 使用 Claude Code Skills 的 AI 自动化开发方案 v2.4
> **参考项目**: superpowers (https://github.com/anthropics/superpowers)

---

## 1. 概述

本文档详细定义了 Cadence 插件的完整配置体系，包括插件注册、市场信息、依赖关系、Hooks 和 Subagent 定义。

### 1.1 配置文件清单

| 文件 | 用途 | 必需性 |
|------|------|-------|
| `plugin.json` | Skill 注册和插件元数据 | ✅ 必须 |
| `marketplace.json` | 市场展示信息 | ✅ 必须 |
| `dependencies.json` | Skill 依赖关系 | ✅ 必须 |
| `hooks.json` | Hooks 配置 | ⭐ 推荐 |
| `agents.json` | Subagent 定义（已废弃） | ❌ 不使用 |

### 1.2 配置目录结构

```
cadence-skills/
├── .claude-plugin/
│   ├── plugin.json              ✅ 插件注册
│   ├── marketplace.json         ✅ 市场信息
│   ├── dependencies.json        ✅ 依赖关系
│   └── hooks.json               ⭐ Hooks配置
│   # agents.json 已废弃 - Subagent 使用 Task tool 动态调用
│
├── skills/                      # Skills目录
├── commands/                    # 命令定义
└── hooks/                       # Hooks脚本
    ├── session-start
    ├── task-complete
    └── run-hook.cmd
```

### 2.2 配置说明

#### 元数据字段

| 字段 | 说明 | 必需 |
|------|------|------|
| `schema_version` | 配置文件版本 | ✅ |
| `name` | 插件名称（唯一标识） | ✅ |
| `version` | 版本号（语义化版本） | ✅ |
| `description` | 简短描述 | ✅ |
| `author` | 作者信息 | ✅ |
| `homepage` | 项目主页 | ⭐ |
| `repository` | 代码仓库 | ⭐ |
| `bugs` | 问题跟踪 | ⭐ |
| `license` | 许可证 | ✅ |
| `keywords` | 关键词（便于搜索） | ⭐ |
| `requirements` | 依赖要求 | ⭐ |

#### Skill 定义字段

| 字段 | 说明 | 必需 |
|------|------|------|
| `name` | Skill 名称（唯一） | ✅ |
| `path` | SKILL.md 文件路径 | ✅ |
| `description` | 简短描述 | ✅ |
| `category` | 分类（meta/prerequisite/node/flow/support） | ⭐ |
| `node_number` | 节点编号（仅节点 Skill） | ⭐ |

#### Command 定义字段

| 字段 | 说明 | 必需 |
|------|------|------|
| `name` | 命令名称（/cadence:xxx） | ✅ |
| `description` | 命令描述 | ✅ |
| `skill` | 关联的 Skill 名称 | ✅ |

## 6. agents.json - Subagent 定义（已废弃）

> ⚠️ **重要**：此配置已废弃。Subagent 应该通过 Task tool 动态调用，不需要在 `agents.json` 中预定义。
>
> **背景**：
> - Phase 1 (P0) 修改已将 Subagent 调用方式从静态配置改为动态 Task tool 调用
> - Claude Code 的 Subagent 不需要预定义，可以直接通过 Task tool 使用内联 prompt 调用
>
> **新的调用方式**：
>
> ### 6.1 使用 Task tool 动态调用 Subagent
>
> **参考文档**：`2026-02-26_Skill_Subagent_Development_v1.0.md` - "调用 Implementer Subagent" 部分
>
> **示例**：
>
> ```markdown
> Task tool (general-purpose):
>   description: "Implement {task_name}"
>   prompt: |
>     You are implementing: **{task_name}**
>
>     ## Task Description
>     {完整任务描述，来自Plan}
>
>     ## Context
>     - **Project**: {project_name}
>     - **Feature Branch**: {feature_branch}
>     - **Work Directory**: {worktree_path}
>     - **Dependencies**: {依赖任务列表}
>     - **Priority**: {P0/P1/P2}
>
>     ## Acceptance Criteria
>     {验收标准列表，来自Plan}
>
>     ## TDD Workflow (MUST FOLLOW)
>     [详细的RED-GREEN-BLUE流程说明]
>
>     ## Output Format
>     [输出格式要求]
> ```
>
> ### 6.2 为什么废弃 agents.json
>
> **问题**：
> - ❌ 静态配置不灵活，需要提前定义所有 Subagent
> - ❌ 无法根据任务动态调整 Subagent 的 prompt
> - ❌ 增加了配置文件的数量和维护成本
>
> **解决方案**：
> - ✅ 使用 Task tool 动态调用 Subagent
> - ✅ 使用内联 prompt，可以根据任务自定义内容
> - ✅ 减少配置文件数量，降低维护成本
>
> ### 6.3 迁移指南
>
> **如果你之前使用 agents.json**：
>
> 1. **删除 `agents.json` 文件**（或保留但不使用）
> 2. **在 Skill 文档中直接使用 Task tool 调用**
>    - 参考：`2026-02-26_Skill_Subagent_Development_v1.0.md`
>    - 参考：`8.1_implementer.md`、`8.2_spec-reviewer.md`、`8.3_code-quality-reviewer.md`
> 3. **使用内联 prompt 传递任务描述、上下文、验收标准**
>

```

### 8.2 版本管理

**语义化版本**：
- **MAJOR**: 不兼容的 API 变更
- **MINOR**: 向后兼容的功能新增
- **PATCH**: 向后兼容的问题修复

**示例**：
- `2.4.0` → 新增 2 个前置 Skill（MINOR）
- `2.4.1` → 修复 bug（PATCH）
- `3.0.0` → 重大架构变更（MAJOR）

### 8.3 依赖管理

**依赖声明原则**：
1. **最小依赖**：只依赖必需的 Skill
2. **明确版本**：指定最低版本要求
3. **可选分离**：区分必需和可选依赖
4. **循环检测**：避免循环依赖
