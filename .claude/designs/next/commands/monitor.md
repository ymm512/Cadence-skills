# /monitor - 实时监控

## 使用场景

实时监控项目进度，自动更新任务状态和进度变化。

## 功能描述

实时监控，包括：
- 实时任务状态更新
- 进度变化提醒
- 警告和阻塞提醒
- 自动刷新（可选）

## 命令参数

```bash
/monitor [--refresh <seconds> | --once]
```

- `--refresh <seconds>` - 自动刷新间隔（默认：5秒）
- `--once` - 只显示一次，不自动刷新

## 使用示例

### 持续监控（每5秒刷新）

```bash
/monitor
```

### 自定义刷新间隔（每10秒）

```bash
/monitor --refresh 10
```

### 只显示一次

```bash
/monitor --once
```

## 监控功能

### 1. 实时任务状态

**监控内容**:
- 当前任务
- 任务状态（in_progress、pending、completed、failed）
- 开始时间
- 已耗时
- 重试次数

**更新频率**: 实时（每次刷新）

---

### 2. 进度变化提醒

**监控内容**:
- 节点完成
- 任务完成
- 测试通过/失败
- 审查通过/失败

**提醒方式**:
- ✅ 绿色 - 成功
- ⚠️ 黄色 - 警告
- ❌ 红色 - 失败

---

### 3. 警告和阻塞提醒

**警告类型**:

| 警告类型 | 触发条件 | 建议操作 |
|---------|---------|---------|
| **测试失败** | 测试失败 | 查看失败日志，修复代码 |
| **审查失败** | 审查失败 | 查看审查报告，修复问题 |
| **重试耗尽** | 重试次数达到上限 | 人工介入 |
| **任务超时** | 任务耗时超过预估 | 人工介入 |
| **依赖阻塞** | 前置任务未完成 | 等待或调整计划 |

**阻塞提醒**:
```
⚠️  阻塞链：
Task 3（失败）
  ↓ 阻塞
Task 4, Task 5（等待）
```

---

### 4. 性能指标

**监控指标**:
- 任务完成率
- 测试通过率
- 审查通过率
- 平均覆盖率
- 平均任务耗时

**更新频率**: 每次刷新

---

### 5. 预估完成时间

**计算逻辑**:
```
预估完成时间 = 当前时间 + (剩余任务数 × 平均任务耗时)
```

**更新频率**: 每次刷新

---

## 自动刷新

### 默认行为

- **刷新间隔**: 5 秒
- **自动更新**: 启用
- **退出方式**: Ctrl+C

### 自定义刷新间隔

```bash
/monitor --refresh 10  # 每10秒刷新
```

### 禁用自动刷新

```bash
/monitor --once  # 只显示一次
```

## 事件日志

### 事件类型

| 事件类型 | 图标 | 说明 |
|---------|------|------|
| **任务开始** | 🔄 | 任务从 pending → in_progress |
| **任务完成** | ✅ | 任务从 in_progress → completed |
| **任务失败** | ❌ | 任务从 in_progress → failed |
| **测试通过** | ✅ | 测试成功 |
| **测试失败** | ⚠️ | 测试失败 |
| **审查通过** | ✅ | 审查成功 |
| **审查失败** | ⚠️ | 审查失败 |
| **覆盖率达标** | ✅ | 覆盖率 ≥ 80% |
| **覆盖率不足** | ⚠️ | 覆盖率 < 80% |

### 事件时间范围

- **默认显示**: 最近 5 分钟
- **可配置**: 最近 N 分钟（未来功能）
